COMPONENT DESIGN PHILOSOPHY:
─────────────────────────────
1. ATOMIC DESIGN PATTERN
   - Atoms: Button, Input, Card (basic building blocks)
   - Molecules: QuizCard, StatsCard (combinations)
   - Organisms: Header, LeaderboardTable (complex UI sections)
   - Templates: Page layouts
   - Pages: Full routes (DashboardPage, QuizPage)

2. SINGLE RESPONSIBILITY
   Each component does ONE thing well:
   - Button.jsx → renders a button with variants
   - QuizCard.jsx → displays quiz preview
   - Header.jsx → top navigation only

3. PROPS VS STATE
   ┌────────────────────────────────────────────────────────────────────────────┐
   │ PROPS (External)                   │ STATE (Internal)                      │
   ├────────────────────────────────────┼───────────────────────────────────────┤
   │ - Passed from parent               │ - Managed within component            │
   │ - Read-only (immutable)            │ - Can be changed with setState        │
   │ - Used for configuration           │ - Used for interactivity              │
   │ Example: <Button variant="primary">│ Example: const [open, setOpen] = ...  │
   └────────────────────────────────────┴───────────────────────────────────────┘

4. COMPOSITION OVER INHERITANCE
   React doesn't use class inheritance. Instead, we compose:
   
   <Card>
       <CardHeader>Title</CardHeader>
       <CardBody>Content</CardBody>
       <CardFooter><Button>Action</Button></CardFooter>
   </Card>

                                    📁 PART 3: STATE MANAGEMENT
════════════════════════════════════════════════════════════════════════════════════════════════════════

THREE TYPES OF STATE IN BRAINSPARK:
────────────────────────────────────

1. LOCAL STATE (useState)
   - Component-specific data
   - Example: form input values, modal open/close
   
   const [searchQuery, setSearchQuery] = useState('');
   const [selectedAnswer, setSelectedAnswer] = useState(null);

2. GLOBAL STATE (Context API)
   - Shared across many components
   - Example: user data, theme preference
   
   ┌─────────────────────────────────────────────────────────────────────────────┐
   │ // context/AuthContext.jsx                                                  │
   │                                                                             │
   │ const AuthContext = createContext(null);                                    │
   │                                                                             │
   │ export const AuthProvider = ({ children }) => {                             │
   │     const [user, setUser] = useState(null);                                 │
   │     const [isAuthenticated, setIsAuthenticated] = useState(false);          │
   │     const [loading, setLoading] = useState(true);                           │
   │                                                                             │
   │     // Check for existing token on mount                                    │
   │     useEffect(() => {                                                       │
   │         const token = localStorage.getItem('token');                        │
   │         if (token) {                                                        │
   │             fetchUser(token);                                               │
   │         } else {                                                            │
   │             setLoading(false);                                              │
   │         }                                                                   │
   │     }, []);                                                                 │
   │                                                                             │
   │     const login = async (email, password) => {                              │
   │         const response = await api.post('/auth/login', { email, password });│
   │         localStorage.setItem('token', response.data.token);                 │
   │         setUser(response.data.user);                                        │
   │         setIsAuthenticated(true);                                           │
   │     };                                                                      │
   │                                                                             │
   │     const logout = () => {                                                  │
   │         localStorage.removeItem('token');                                   │
   │         setUser(null);                                                      │
   │         setIsAuthenticated(false);                                          │
   │     };                                                                      │
   │                                                                             │
   │     return (                                                                │
   │         <AuthContext.Provider value={{                                      │
   │             user, isAuthenticated, loading, login, logout                   │
   │         }}>                                                                 │
   │             {children}                                                      │
   │         </AuthContext.Provider>                                             │
   │     );                                                                      │
   │ };                                                                          │
   │                                                                             │
   │ // Custom hook for easy usage                                               │
   │ export const useAuth = () => useContext(AuthContext);                       │
   └─────────────────────────────────────────────────────────────────────────────┘

3. SERVER STATE (API Data)
   - Data from backend
   - Fetched with useEffect + useState
   - Example: quizzes list, leaderboard data
   
   ┌─────────────────────────────────────────────────────────────────────────────┐
   │ const [quizzes, setQuizzes] = useState([]);                                 │
   │ const [loading, setLoading] = useState(true);                               │
   │ const [error, setError] = useState(null);                                   │
   │                                                                             │
   │ useEffect(() => {                                                           │
   │     const fetchQuizzes = async () => {                                      │
   │         try {                                                               │
   │             setLoading(true);                                               │
   │             const response = await api.get('/quiz');                        │
   │             setQuizzes(response.data.data);                                 │
   │         } catch (err) {                                                     │
   │             setError(err.message);                                          │
   │         } finally {                                                         │
   │             setLoading(false);                                              │
   │         }                                                                   │
   │     };                                                                      │
   │     fetchQuizzes();                                                         │
   │ }, []);                                                                     │
   └─────────────────────────────────────────────────────────────────────────────┘


WHY CONTEXT API AND NOT REDUX?
──────────────────────────────
┌────────────────────────────────────────────────────────────────────────────────────┐
│ Context API                           │ Redux                                      │
├───────────────────────────────────────┼────────────────────────────────────────────┤
│ ✅ Built into React                   │ ❌ External library                        │
│ ✅ Simple for small-medium apps       │ ✅ Better for large apps                   │
│ ✅ Less boilerplate                   │ ❌ Actions, reducers, store setup          │
│ ✅ Easy to understand                 │ ✅ DevTools, time-travel debugging         │
│ ❌ Can cause re-renders               │ ✅ Optimized for performance               │
└───────────────────────────────────────┴────────────────────────────────────────────┘

For BrainSpark's scale, Context API is perfect!

CORE HOOKS USED IN BRAINSPARK:
──────────────────────────────

1️⃣ useState - Managing Local State
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Purpose: Store and update component-level data

┌─────────────────────────────────────────────────────────────────────────────────────┐
│ // Syntax                                                                           │
│ const [state, setState] = useState(initialValue);                                   │
│                                                                                     │
│ // Examples in BrainSpark:                                                          │
│                                                                                     │
│ // Simple value                                                                     │
│ const [currentQuestion, setCurrentQuestion] = useState(0);                          │
│                                                                                     │
│ // Boolean                                                                          │
│ const [isModalOpen, setIsModalOpen] = useState(false);                              │
│                                                                                     │
│ // Object                                                                           │
│ const [formData, setFormData] = useState({                                          │
│     email: '',                                                                      │
│     password: '',                                                                   │
│     rememberMe: false                                                               │
│ });                                                                                 │
│                                                                                     │
│ // Array                                                                            │
│ const [selectedAnswers, setSelectedAnswers] = useState([]);                         │
│                                                                                     │
│ // Updating object state (must spread previous state)                               │
│ setFormData(prev => ({ ...prev, email: 'new@email.com' }));                         │
│                                                                                     │
│ // Updating array state                                                             │
│ setSelectedAnswers(prev => [...prev, newAnswer]);                                   │
│                                                                                     │
│ // Lazy initialization (expensive computation)                                      │
│ const [data, setData] = useState(() => expensiveComputation());                     │
└─────────────────────────────────────────────────────────────────────────────────────┘

VIVA QUESTION: "Why do we spread previous state when updating objects?"
ANSWER: "useState doesn't merge objects automatically like this.setState in class 
components. If we don't spread, we lose all other properties. Spreading creates a 
new object reference, which triggers a re-render."


2️⃣ useEffect - Side Effects & Lifecycle
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Purpose: Handle side effects - API calls, subscriptions, DOM manipulation

┌─────────────────────────────────────────────────────────────────────────────────────┐
│ // CASE 1: Run on every render (no dependency array)                                │
│ useEffect(() => {                                                                   │
│     console.log('Runs after every render');                                         │
│ });                                                                                 │
│                                                                                     │
│ // CASE 2: Run only once on mount (empty dependency array)                          │
│ useEffect(() => {                                                                   │
│     fetchQuizzes();   // API call on component mount                                │
│ }, []);                                                                             │
│                                                                                     │
│ // CASE 3: Run when specific dependency changes                                     │
│ useEffect(() => {                                                                   │
│     fetchQuizById(quizId);                                                          │
│ }, [quizId]);         // Re-runs when quizId changes                                │
│                                                                                     │
│ // CASE 4: Cleanup function (for subscriptions, timers)                             │
│ useEffect(() => {                                                                   │
│     const timer = setInterval(() => {                                               │
│         setTimeLeft(prev => prev - 1);                                              │
│     }, 1000);                                                                       │
│                                                                                     │
│     return () => {                                                                  │
│         clearInterval(timer);  // Cleanup on unmount                                │
│     };                                                                              │
│ }, []);                                                                             │
└─────────────────────────────────────────────────────────────────────────────────────┘

REAL EXAMPLE - Timer in QuizPage:
┌─────────────────────────────────────────────────────────────────────────────────────┐
│ useEffect(() => {                                                                   │
│     // Only run timer in timed mode                                                 │
│     if (mode !== 'timed') return;                                                   │
│                                                                                     │
│     const timer = setInterval(() => {                                               │
│         setTimeLeft(prev => {                                                       │
│             if (prev <= 1) {                                                        │
│                 handleTimeUp();          // Auto-submit when time runs out          │
│                 return 30;                // Reset for next question                │
│             }                                                                       │
│             return prev - 1;                                                        │
│         });                                                                         │
│     }, 1000);                                                                       │
│                                                                                     │
│     return () => clearInterval(timer);   // Cleanup                                 │
│ }, [mode, currentQuestion]);             // Reset timer on question change          │
└─────────────────────────────────────────────────────────────────────────────────────┘

VIVA QUESTION: "What happens if you don't provide cleanup?"
ANSWER: "Memory leaks! The interval keeps running even after component unmounts, 
causing errors and wasted resources. Cleanup function runs before unmount and 
before every re-run of the effect."


3️⃣ useContext - Consuming Context
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Purpose: Access global state without prop drilling

┌─────────────────────────────────────────────────────────────────────────────────────┐
│ // Creating Context                                                                 │
│ const AuthContext = createContext(null);                                            │
│                                                                                     │
│ // Provider Component                                                               │
│ export const AuthProvider = ({ children }) => {                                     │
│     const [user, setUser] = useState(null);                                         │
│     return (                                                                        │
│         <AuthContext.Provider value={{ user, setUser }}>                            │
│             {children}                                                              │
│         </AuthContext.Provider>                                                     │
│     );                                                                              │
│ };                                                                                  │
│                                                                                     │
│ // Custom hook for consumption                                                      │
│ export const useAuth = () => {                                                      │
│     const context = useContext(AuthContext);                                        │
│     if (!context) {                                                                 │
│         throw new Error('useAuth must be used within AuthProvider');                │
│     }                                                                               │
│     return context;                                                                 │
│ };                                                                                  │
│                                                                                     │
│ // Using in any component                                                           │
│ const ProfilePage = () => {                                                         │
│     const { user, logout } = useAuth();                                             │
│     return <h1>Welcome, {user.name}</h1>;                                           │

App → Layout → Sidebar → UserCard → user

WITH CONTEXT:

4️⃣ useRef - Mutable References
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Purpose: Store mutable values that don't trigger re-renders, or access DOM elements

┌─────────────────────────────────────────────────────────────────────────────────────┐
│ // DOM Reference                                                                    │
│ const inputRef = useRef(null);                                                      │
│                                                                                     │
│ useEffect(() => {                                                                   │
│     inputRef.current.focus();  // Focus input on mount                              │
│ }, []);                                                                             │
│                                                                                     │
│ return <input ref={inputRef} />;                                                    │
│                                                                                     │
│ // Mutable value that persists across renders                                       │
│ const intervalRef = useRef(null);                                                   │
│                                                                                     │
│ const startTimer = () => {                                                          │
│     intervalRef.current = setInterval(() => {                                       │
│         // ...                                                                      │
│     }, 1000);                                                                       │
│ };                                                                                  │
│                                                                                     │
│ const stopTimer = () => {                                                           │
│     clearInterval(intervalRef.current);                                             │
│ };                                                                                  │
│                                                                                     │
│ // Previous value tracking                                                          │
│ const prevValueRef = useRef();                                                      │
│ useEffect(() => {                                                                   │
│     prevValueRef.current = currentValue;                                            │
│ });                                                                                 │
│ const prevValue = prevValueRef.current;                                             │
└─────────────────────────────────────────────────────────────────────────────────────┘

VIVA QUESTION: "Difference between useRef and useState?"
ANSWER: "useRef changes don't trigger re-renders, useState changes do. useRef gives 
a mutable .current property that persists across renders. Use useRef for DOM access 
or values you need to persist but don't need to display."


5️⃣ useMemo - Memoize Expensive Computations
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Purpose: Cache computed values to avoid recalculating on every render

┌─────────────────────────────────────────────────────────────────────────────────────┐
│ // Without useMemo - recalculates every render!                                     │
│ const filteredQuestions = questions.filter(q =>                                     │
│     q.subject.includes(searchQuery) ||                                              │
│     q.text.includes(searchQuery)                                                    │
│ );                                                                                  │
│                                                                                     │
│ // With useMemo - only recalculates when dependencies change                        │
│ const filteredQuestions = useMemo(() => {                                           │
│     return questions.filter(q =>                                                    │
│         q.subject.includes(searchQuery) ||                                          │
│         q.text.includes(searchQuery)                                                │
│     );                                                                              │
│ }, [questions, searchQuery]);                                                       │
│                                                                                     │
│ // Real example in BookmarksPage                                                    │
│ const filteredBookmarks = useMemo(() => {                                           │
│     return bookmarks.filter(q => {                                                  │
│         const matchesSearch = q.question.toLowerCase()                              │
│             .includes(searchQuery.toLowerCase());                                   │
│         const matchesSubject = selectedSubject === 'All' ||                         │
│             q.subject === selectedSubject;                                          │
│         return matchesSearch && matchesSubject;                                     │
│     });                                                                             │
│ }, [bookmarks, searchQuery, selectedSubject]);                                      │
└─────────────────────────────────────────────────────────────────────────────────────┘


6️⃣ useCallback - Memoize Functions
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Purpose: Cache function references to prevent unnecessary re-renders of child components

┌─────────────────────────────────────────────────────────────────────────────────────┐
│ // Problem: New function created every render, child always re-renders             │
│ const handleClick = () => {                                                         │
│     setCount(count + 1);                                                            │
│ };                                                                                  │
│                                                                                     │
│ // Solution: Memoized function, only changes when dependencies change               │
│ const handleClick = useCallback(() => {                                             │
│     setCount(prev => prev + 1);                                                     │
│ }, []);  // Empty = never changes                                                   │
│                                                                                     │
│ // With dependencies                                                                │
│ const handleSubmit = useCallback(() => {                                            │
│     submitQuiz(quizId, answers);                                                    │
│ }, [quizId, answers]);                                                              │
│                                                                                     │
│ // Useful when passing callbacks to optimized child components                      │
│ <MemoizedButton onClick={handleClick}>Click</MemoizedButton>                        │
└─────────────────────────────────────────────────────────────────────────────────────┘

VIVA QUESTION: "When should you use useMemo vs useCallback?"
ANSWER: "useMemo memoizes the RESULT of a function call (computed value). 
useCallback memoizes the FUNCTION ITSELF (function reference). Use useMemo for 
expensive calculations, useCallback for functions passed to optimized children."

1️⃣ useLocalStorage - Persist State
┌─────────────────────────────────────────────────────────────────────────────────────┐
│ // hooks/useLocalStorage.js                                                         │
│                                                                                     │
│ import { useState, useEffect } from 'react';                                        │
│                                                                                     │
│ const useLocalStorage = (key, initialValue) => {                                    │
│     // Get initial value from localStorage or use default                           │
│     const [storedValue, setStoredValue] = useState(() => {                          │
│         try {                                                                       │
│             const item = localStorage.getItem(key);                                 │
│             return item ? JSON.parse(item) : initialValue;                          │
│         } catch (error) {                                                           │
│             console.error(error);                                                   │
│             return initialValue;                                                    │
│         }                                                                           │
│     });                                                                             │
│                                                                                     │
│     // Update localStorage when value changes                                       │
│     useEffect(() => {                                                               │
│         try {                                                                       │
│             localStorage.setItem(key, JSON.stringify(storedValue));                 │
│         } catch (error) {                                                           │
│             console.error(error);                                                   │
│         }                                                                           │
│     }, [key, storedValue]);                                                         │
│                                                                                     │
│     return [storedValue, setStoredValue];                                           │
│ };                                                                                  │
│                                                                                     │
│ export default useLocalStorage;                                                     │
│                                                                                     │
│ // USAGE:                                                                           │
│ const [theme, setTheme] = useLocalStorage('theme', 'dark');                         │
│ const [soundEnabled, setSoundEnabled] = useLocalStorage('sound', true);             │
└─────────────────────────────────────────────────────────────────────────────────────┘


2️⃣ useBookmarks - Bookmark Questions
┌─────────────────────────────────────────────────────────────────────────────────────┐
│ // hooks/useBookmarks.js                                                            │
│                                                                                     │
│ import useLocalStorage from './useLocalStorage';                                    │
│                                                                                     │
│ const useBookmarks = () => {                                                        │
│     const [bookmarks, setBookmarks] = useLocalStorage('bookmarkedQuestions', []);   │
│                                                                                     │
│     const addBookmark = (question) => {                                             │
│         setBookmarks(prev => {                                                      │
│             if (prev.find(q => q.id === question.id)) return prev;                  │
│             return [...prev, { ...question, bookmarkedAt: new Date().toISOString() }];│
│         });                                                                         │
│     };                                                                              │
│                                                                                     │
│     const removeBookmark = (questionId) => {                                        │
│         setBookmarks(prev => prev.filter(q => q.id !== questionId));                │
│     };                                                                              │
│                                                                                     │
│     const isBookmarked = (questionId) => {                                          │
│         return bookmarks.some(q => q.id === questionId);                            │
│     };                                                                              │
│                                                                                     │
│     const toggleBookmark = (question) => {                                          │
│         if (isBookmarked(question.id)) {                                            │
│             removeBookmark(question.id);                                            │
│         } else {                                                                    │
│             addBookmark(question);                                                  │
│         }                                                                           │
│     };                                                                              │
│                                                                                     │
│     return { bookmarks, addBookmark, removeBookmark, isBookmarked, toggleBookmark };│
│ };                                                                                  │
│                                                                                     │
│ export default useBookmarks;                                                        │
└─────────────────────────────────────────────────────────────────────────────────────┘


3️⃣ useSound - Sound Effects
┌─────────────────────────────────────────────────────────────────────────────────────┐
│ // hooks/useSound.js                                                                │

│ import { useCallback } from 'react';                                                │
│ import useLocalStorage from './useLocalStorage';                                    │
│                                                                                     │
│ const sounds = {                                                                    │
│     correct: '/sounds/correct.mp3',                                                 │
│     wrong: '/sounds/wrong.mp3',                                                     │
│     click: '/sounds/click.mp3',                                                     │

4️⃣ useQuiz - Quiz Logic
┌─────────────────────────────────────────────────────────────────────────────────────┐
│ // hooks/useQuiz.js                                                                 │
│                                                                                     │
│ const useQuiz = (quizId) => {                                                       │
│     const [quiz, setQuiz] = useState(null);                                         │
│     const [currentQuestion, setCurrentQuestion] = useState(0);                      │
│     const [answers, setAnswers] = useState([]);                                     │
│     const [loading, setLoading] = useState(true);                                   │
│     const [error, setError] = useState(null);                                       │
│                                                                                     │
│     // Fetch quiz on mount                                                          │
│     useEffect(() => {                                                               │
│         const fetchQuiz = async () => {                                             │
│             try {                                                                   │
│                 const response = await api.get(`/quiz/${quizId}`);                  │
│                 setQuiz(response.data.data);                                        │
│             } catch (err) {                                                         │
│                 setError(err.message);                                              │
│             } finally {                                                             │
│                 setLoading(false);                                                  │
│             }                                                                       │
│         };                                                                          │
│         fetchQuiz();                                                                │
│     }, [quizId]);                                                                   │
│                                                                                     │
│     const selectAnswer = (optionIndex) => {                                         │
│         setAnswers(prev => {                                                        │
│             const newAnswers = [...prev];                                           │
│             newAnswers[currentQuestion] = optionIndex;                              │
│             return newAnswers;                                                      │
│         });                                                                         │
│     };                                                                              │
│                                                                                     │
│     const nextQuestion = () => {                                                    │
│         if (currentQuestion < quiz.questions.length - 1) {                          │
│             setCurrentQuestion(prev => prev + 1);                                   │
│         }                                                                           │
│     };                                                                              │
│                                                                                     │
│     const prevQuestion = () => {                                                    │
│         if (currentQuestion > 0) {                                                  │
│             setCurrentQuestion(prev => prev - 1);                                   │
│         }                                                                           │
│     };                                                                              │
│                                                                                     │
│     const calculateScore = () => {                                                  │
│         return answers.reduce((score, answer, index) => {                           │
│             return score + (answer === quiz.questions[index].correctAnswer ? 1 : 0);│
│         }, 0);                                                                      │
│     };                                                                              │
│                                                                                     │
│     return {                                                                        │
│         quiz, loading, error,                                                       │
│         currentQuestion, answers,                                                   │
│         selectAnswer, nextQuestion, prevQuestion, calculateScore                    │
│     };                                                                              │
│ };                                                                                  │
└─────────────────────────────────────────────────────────────────────────────────────┘

VIVA QUESTION: "Why create custom hooks?"
ANSWER: "Custom hooks extract reusable logic that can be shared across components. 
They start with 'use' prefix and can use other hooks. Benefits: DRY code, separation 
of concerns, easier testing, cleaner components."

                                    📁 PART 7: ANIMATIONS WITH FRAMER MOTION
════════════════════════════════════════════════════════════════════════════════════════════════════════

WHAT IS FRAMER MOTION?
──────────────────────
Framer Motion is a production-ready motion library for React. It provides:
- Declarative animations with simple props
- Gesture support (drag, hover, tap)
- Layout animations
- AnimatePresence for exit animations
- Spring and keyframe animations

CORE CONCEPTS:
──────────────

1️⃣ motion Components
┌─────────────────────────────────────────────────────────────────────────────────────┐
│ import { motion } from 'framer-motion';                                             │
│                                                                                     │
│ // motion.div, motion.span, motion.button, motion.ul, etc.                          │
│ // Any HTML element can become animated by prefixing with motion.                   │
│                                                                                     │
│ <motion.div                                                                         │
│     initial={{ opacity: 0 }}        // Starting state                               │
│     animate={{ opacity: 1 }}        // End state                                    │
│     transition={{ duration: 0.5 }}  // How to animate                               │
│ >                                                                                   │
│     Hello World                                                                     │
│ </motion.div>                                                                       │
└─────────────────────────────────────────────────────────────────────────────────────┘


2️⃣ Animation Properties
┌─────────────────────────────────────────────────────────────────────────────────────┐
│ <motion.div                                                                         │
│     // INITIAL - where animation starts                                             │
│     initial={{ opacity: 0, y: -20, scale: 0.9 }}                                    │
│                                                                                     │
│     // ANIMATE - where animation ends                                               │
│     animate={{ opacity: 1, y: 0, scale: 1 }}                                        │
│                                                                                     │
│     // EXIT - animation when component unmounts (needs AnimatePresence)             │
│     exit={{ opacity: 0, y: 20 }}                                                    │
│                                                                                     │
│     // TRANSITION - controls timing                                                 │
│     transition={{                                                                   │
│         duration: 0.4,                    // seconds                                │
│         delay: 0.1,                       // wait before starting                   │
│         ease: "easeOut",                  // easing function                        │
│         type: "spring",                   // spring physics                         │
│         stiffness: 300,                   // spring tension                         │
│         damping: 20                       // spring friction                        │
│     }}                                                                              │
│                                                                                     │
│     // HOVER animation                                                              │
│     whileHover={{ scale: 1.05, boxShadow: "0 10px 30px rgba(0,0,0,0.2)" }}          │
│                                                                                     │
│     // TAP/CLICK animation                                                          │
│     whileTap={{ scale: 0.95 }}                                                      │
│ />                                                                                  │
└─────────────────────────────────────────────────────────────────────────────────────┘


3️⃣ AnimatePresence - Exit Animations
┌─────────────────────────────────────────────────────────────────────────────────────┐
│ import { AnimatePresence, motion } from 'framer-motion';                            │
│                                                                                     │
│ // AnimatePresence wraps elements that may be conditionally rendered                │
│ // It allows exit animations before component unmounts                              │
│                                                                                     │

│             key="modal"                   // Key is REQUIRED for AnimatePresence    │
│             initial={{ opacity: 0 }}                                                │
│             animate={{ opacity: 1 }}                                                │
│             exit={{ opacity: 0 }}         // Plays when component unmounts          │
│         >                                                                           │
│             <Modal />                                                               │
│         </motion.div>                                                               │
│     )}                                                                              │
│ </AnimatePresence>                                                                  │
└─────────────────────────────────────────────────────────────────────────────────────┘

└─────────────────────────────────────────────────────────────────────────────────────┘

VIVA QUESTION: "Why use Framer Motion instead of CSS animations?"

                                    📁 PART 8: PORTALS (ReactDOM.createPortal)
════════════════════════════════════════════════════════════════════════════════════════════════════════

WHAT IS A PORTAL?
─────────────────
A Portal lets you render children into a DOM node that exists OUTSIDE the parent 
component's DOM hierarchy, while still maintaining React's component tree structure.

WHY USE PORTALS?
────────────────
┌─────────────────────────────────────────────────────────────────────────────────────┐
│ PROBLEM: Modal inside a component with overflow: hidden or z-index issues          │
│                                                                                     │
│ <div style={{ overflow: 'hidden', position: 'relative' }}>                          │
│     <SomeComponent />                                                               │
│     <Modal />  ← Modal gets clipped or stuck behind other elements!                │
│ </div>                                                                              │
│                                                                                     │
│ SOLUTION: Portal renders Modal to document.body                                     │
│                                                                                     │
│ DOM Tree:                                                                           │
│ <body>                                                                              │
│     <div id="root">                                                                 │
│         <App>                                                                       │
│             <Page>                                                                  │
│                 <SomeComponent />                                                   │
│                 {/* Modal is NOT here in DOM */}                                    │
│             </Page>                                                                 │
│         </App>                                                                      │
│     </div>                                                                          │
│     <div class="modal-overlay">  ← Modal renders here via Portal!                  │
│         <div class="modal">...</div>                                                │
│     </div>                                                                          │
│ </body>                                                                             │
└─────────────────────────────────────────────────────────────────────────────────────┘

AXIOS SETUP & CONFIGURATION:

┌─────────────────────────────────────────────────────────────────────────────────────┐
│ // utils/api.js                                                                     │
│                                                                                     │
│ import axios from 'axios';                                                          │
│                                                                                     │
│ // Create axios instance with base configuration                                    │
│ const api = axios.create({                                                          │
│     baseURL: import.meta.env.VITE_API_URL || 'http://localhost:5000/api',           │
│     timeout: 10000,                        // 10 second timeout                     │
│     headers: {                                                                      │
│         'Content-Type': 'application/json'                                          │
│     }                                                                               │
│ });                                                                                 │
│                                                                                     │
│ // REQUEST INTERCEPTOR - runs before every request                                  │
│ api.interceptors.request.use(                                                       │
│     (config) => {                                                                   │
│         // Attach token to every request                                            │
│         const token = localStorage.getItem('token');                                │
│         if (token) {                                                                │
│             config.headers.Authorization = `Bearer ${token}`;                       │
│         }                                                                           │
│         return config;                                                              │
│     },                                                                              │
│     (error) => {                                                                    │
│         return Promise.reject(error);                                               │
│     }                                                                               │
│ );                                                                                  │
│                                                                                     │
│ // RESPONSE INTERCEPTOR - runs after every response                                 │
│ api.interceptors.response.use(                                                      │
│     (response) => {                                                                 │
│         // Success - just return response                                           │
│         return response;                                                            │
│     },                                                                              │
│     (error) => {                                                                    │
│         // Handle 401 Unauthorized - token expired/invalid                          │
│         if (error.response?.status === 401) {                                       │
│             localStorage.removeItem('token');                                       │
│             window.location.href = '/login';                                        │
│         }                                                                           │
│                                                                                     │
│         // Handle 404 Not Found                                                     │
│         if (error.response?.status === 404) {                                       │
│             console.error('Resource not found');                                    │
│         }                                                                           │
│                                                                                     │
│         // Handle 500 Server Error                                                  │
│         if (error.response?.status >= 500) {                                        │
│             console.error('Server error');                                          │
│         }                                                                           │
│                                                                                     │
│         return Promise.reject(error);                                               │
│     }                                                                               │
│ );                                                                                  │
│                                                                                     │
│ export default api;                                                                 │

WHAT IS AN ERROR BOUNDARY?
──────────────────────────
Error Boundaries are React components that catch JavaScript errors in child components,
log errors, and display a fallback UI instead of crashing the whole app.

┌─────────────────────────────────────────────────────────────────────────────────────┐
│ // components/common/ErrorBoundary.jsx                                              │
│                                                                                     │
│ import React from 'react';                                                          │
│ import './ErrorBoundary.css';                                                       │
│                                                                                     │
│ class ErrorBoundary extends React.Component {                                       │
│     constructor(props) {                                                            │
│         super(props);                                                               │
│         this.state = { hasError: false, error: null, errorInfo: null };             │
│     }                                                                               │
│                                                                                     │
│     // Called when an error is thrown in a descendant                               │
│     static getDerivedStateFromError(error) {                                        │
│         // Update state to show fallback UI                                         │
│         return { hasError: true };                                                  │
│     }                                                                               │
│                                                                                     │
│     // Called after error is caught - good for logging                              │
│     componentDidCatch(error, errorInfo) {                                           │
│         this.setState({ error, errorInfo });                                        │
│                                                                                     │
│         // Log to error reporting service                                           │
│         console.error('Error caught by boundary:', error, errorInfo);               │
│         // Could send to Sentry, LogRocket, etc.                                    │
│     }                                                                               │
│                                                                                     │
│     handleReset = () => {                                                           │
│         this.setState({ hasError: false, error: null, errorInfo: null });           │
│     };                                                                              │
│                                                                                     │
│     render() {                                                                      │
│         if (this.state.hasError) {                                                  │
│             return (                                                                │
│                 <div className="error-boundary">                                    │
│                     <div className="error-boundary__icon">💥</div>                  │
│                     <h2>Oops! Something went wrong</h2>                             │

VIVA QUESTION: "Why must Error Boundaries be class components?"
ANSWER: "Error boundaries require lifecycle methods (componentDidCatch and

WHAT IS VITE?
─────────────
Vite (French for "fast") is a modern build tool for frontend development:
- Ultra-fast dev server with Hot Module Replacement (HMR)
- Native ES modules for instant updates
- Rollup-based production builds
- Out-of-box TypeScript, JSX, CSS support

WHY VITE OVER CREATE-REACT-APP (CRA)?
┌─────────────────────────────────────────────────────────────────────────────────────┐
│                                                                                     │
│ Create React App                        Vite                                        │
│ ──────────────────                      ────                                        │

│ ❌ Webpack (complex config)             ✅ Simple vite.config.js                    │

│ ❌ Outdated (deprecated)                ✅ Actively maintained                     │

VIVA QUESTION: "How does Vite's dev server work differently?"
ANSWER: "Traditional bundlers bundle everything before serving. Vite serves files

REACT PERFORMANCE TECHNIQUES:

1️⃣ React.memo - Prevent Unnecessary Re-renders
┌─────────────────────────────────────────────────────────────────────────────────────┐
│ // Without memo - re-renders on every parent render                                 │
│ const QuizCard = ({ quiz }) => {                                                    │
│     return <div>{quiz.title}</div>;                                                 │
│ };                                                                                  │
│                                                                                     │
│ // With memo - only re-renders when props change                                    │
│ const QuizCard = React.memo(({ quiz }) => {                                         │
│     return <div>{quiz.title}</div>;                                                 │
│ });                                                                                 │
│                                                                                     │
│ // Custom comparison function                                                       │
│ const QuizCard = React.memo(                                                        │
│     ({ quiz }) => <div>{quiz.title}</div>,                                          │
│     (prevProps, nextProps) => prevProps.quiz.id === nextProps.quiz.id               │
│ );                                                                                  │
└─────────────────────────────────────────────────────────────────────────────────────┘


2️⃣ useMemo - Memoize Expensive Calculations
┌─────────────────────────────────────────────────────────────────────────────────────┐
│ // Expensive filtering - runs every render without useMemo                          │
│ const filteredQuizzes = useMemo(() => {                                             │
│     return quizzes                                                                  │
│         .filter(q => q.category === selectedCategory)                               │
│         .sort((a, b) => b.score - a.score)                                          │
│         .slice(0, 10);                                                              │
│ }, [quizzes, selectedCategory]);  // Only recalculate when these change             │
└─────────────────────────────────────────────────────────────────────────────────────┘


3️⃣ useCallback - Memoize Callbacks
┌─────────────────────────────────────────────────────────────────────────────────────┐
│ // Without useCallback - new function every render                                  │
│ const handleClick = (id) => selectQuiz(id);                                         │
│                                                                                     │
│ // With useCallback - stable reference                                              │
│ const handleClick = useCallback((id) => {                                           │
│     selectQuiz(id);                                                                 │
│ }, [selectQuiz]);                                                                   │
└─────────────────────────────────────────────────────────────────────────────────────┘

VIVA QUESTION: "When should you use React.memo?"
ANSWER: "Use React.memo when: 1) Component renders often with same props, 
2) Re-rendering is expensive, 3) Component is pure (same props = same output). 
Avoid for: simple components (overhead not worth it), frequently changing props."

REACT FUNDAMENTALS:
───────────────────

Q1: What is React?
A: React is a JavaScript library for building user interfaces. It uses a component-based 
   architecture, Virtual DOM for efficient updates, and declarative syntax (JSX).

Q2: What is JSX?
A: JSX (JavaScript XML) is a syntax extension that lets us write HTML-like code in 
   JavaScript. It gets compiled to React.createElement() calls by Babel.

Q3: What is the Virtual DOM?
A: Virtual DOM is a lightweight copy of the real DOM in memory. React compares changes 
   between Virtual DOMs (diffing) and updates only changed parts of real DOM (reconciliation).

Q4: Difference between props and state?
A: Props are read-only data passed from parent to child. State is mutable data managed 
   within a component. Props flow down, state is local.

Q5: What is lifting state up?
A: When multiple components need shared state, we move it to their closest common ancestor 
   and pass down via props. This keeps state synchronized.


HOOKS QUESTIONS:
────────────────

Q6: What are React Hooks?
A: Hooks are functions that let you use React features (state, lifecycle) in functional 
   components. Introduced in React 16.8 to replace class component patterns.

Q7: Rules of Hooks?
A: 1) Only call hooks at the top level (not in loops, conditions, nested functions)
   2) Only call hooks from React functions (components or custom hooks)
   3) Hooks must be called in the same order every render

Q8: What is the dependency array in useEffect?
A: It controls when the effect runs:
   - No array: runs after every render
   - Empty []: runs once on mount
   - [dep1, dep2]: runs when dependencies change

Q9: When does useEffect cleanup run?
A: Cleanup runs: 1) Before the component unmounts, 2) Before every re-execution of 
   the effect (when dependencies change). Used for cleaning up subscriptions, timers.

Q10: useMemo vs useCallback?
A: useMemo memoizes a computed VALUE. useCallback memoizes a FUNCTION reference.
    useMemo: const result = useMemo(() => expensiveCalc(), [deps])
    useCallback: const fn = useCallback(() => doSomething(), [deps])


ROUTING QUESTIONS:
──────────────────

Q11: What is client-side routing?
A: Navigation happens in the browser without full page reloads. JavaScript intercepts 
   link clicks, updates URL, and renders appropriate component.

Q12: What is useNavigate?
A: A hook from React Router that returns a function for programmatic navigation.
   const navigate = useNavigate(); navigate('/dashboard');

Q13: What is useParams?
A: A hook that returns an object of URL parameters from dynamic routes.
   Route: /quiz/:id → useParams() returns { id: "value" }

Q14: Protected Route pattern?
A: A wrapper component that checks authentication before rendering children.
   If not authenticated, redirects to login page.


STATE MANAGEMENT:
─────────────────

Q15: What is prop drilling?
A: Passing props through multiple levels of components that don't need the data, 
   just to reach a deeply nested component. Solution: Context API or state libraries.

Q16: When to use Context vs Redux?
A: Context: Simple to medium apps, infrequent updates, no need for middleware.
   Redux: Large apps, frequent updates, need for devtools, time-travel debugging.

Q17: What causes unnecessary re-renders?
A: 1) Parent re-renders (even if child props unchanged)
   2) New object/array references on each render
   3) Context value changes
   Solution: React.memo, useMemo, useCallback


PERFORMANCE:
────────────

Q18: What is code splitting?
A: Dividing code into smaller chunks that load on-demand. Reduces initial bundle size.
   Implemented with React.lazy() and dynamic imports.

Q19: What is tree shaking?
A: Build optimization that removes unused code from the final bundle. Works with ES 
   modules (import/export). Reduces bundle size.

Q20: How to optimize list rendering?
A: 1) Use unique 'key' prop (not index)
   2) Virtualization for large lists (react-window)
   3) Memoize list items
   4) useMemo for filtering/sorting


CSS/STYLING:
────────────

Q21: What is BEM naming?
A: Block__Element--Modifier. A naming convention for CSS classes that improves 
   readability and prevents conflicts. Example: .card__title--highlighted

Q22: Mobile-first design?
A: Write base CSS for mobile, then add media queries for larger screens.
   Opposite of desktop-first (base = desktop, media queries for smaller).

Q23: CSS Variables vs SASS variables?
A: CSS Variables: Runtime, changeable via JS, cascade normally, browser native
   SASS Variables: Compile-time, replaced with values, need preprocessor


BUILD TOOLS:
────────────

Q24: Why Vite over CRA?
A: Vite offers: faster dev server (native ES modules), faster HMR, simpler config,
   smaller dependencies, actively maintained (CRA is deprecated).

Q25: What is HMR?
A: Hot Module Replacement - updates modules in browser without full page reload.
   Preserves application state during development.


TESTING (Bonus):
────────────────

Q26: Types of React testing?
A: 1) Unit tests (individual functions/components)
   2) Integration tests (component interactions)
   3) End-to-end tests (full user flows)
   Tools: Jest, React Testing Library, Cypress

Q27: What is React Testing Library?
A: A library for testing React components that encourages testing from user's 
   perspective (what they see/interact with) rather than implementation details.


COMMON MISTAKES:
────────────────

Q28: Why not use array index as key?
A: Index changes when items are reordered/deleted, causing incorrect component 
   updates, state bugs, and performance issues. Use unique IDs instead.

Q29: Why avoid inline object/function props?
A: Creates new reference every render, causing child re-renders even with React.memo.
   Solution: Define outside component or use useMemo/useCallback.

Q30: Why is my useEffect running infinitely?
A: Probably missing dependency array, or updating a dependency inside the effect.
   Solution: Add proper dependencies, use functional updates (prev => prev + 1).
